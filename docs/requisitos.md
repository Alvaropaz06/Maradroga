# Documento de Requisitos – SportReserve / Pistalia

## 1. Identificación del Problema

Dificultad para gestionar reservas de pistas deportivas y falta de personalización para los usuarios.

---

## 2. Usuarios del Sistema

### Deportistas
- Realizan reservas
- Gestionan pagos

### Administradores
- Gestionan pistas
- Supervisan el sistema

### Usuarios no registrados
- Consultan disponibilidad
- Pueden registrarse

---

## 3. Requisitos Funcionales (RF)

- RF1: Registro de usuarios
- RF2: Inicio de sesión
- RF3: Gestión de pistas (CRUD)
- RF4: Consulta de disponibilidad
- RF5: Reservas por horas
- RF6: Cancelación con 24h
- RF7: Gestión de pagos
- RF8: Recomendaciones por deporte
- RF9: Emails automáticos
- RF10: Gestión de empleados

---

## 4. Requisitos No Funcionales (RNF)

- RNF1: Web responsive
- RNF2: Interfaz intuitiva
- RNF3: Base de datos relacional
- RNF4: Seguridad en login
- RNF5: Buen rendimiento
- RNF6: Uso de GitHub

---

## 5. Estructura de una Reserva

- Usuario (ID)
- Pista (ID, ubicación)
- Fecha y hora
- Método de pago
- Estado del pago

---

## 6. Objetivo del Sistema

Desarrollar una aplicación web para gestionar reservas deportivas de forma eficiente.
